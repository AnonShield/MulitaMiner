# Dataset Summary

This document provides a detailed overview, analysis, and results of the MulitaMiner dataset v1.0. All vulnerabilities were extracted using the DeepSeek language model, which processed each OpenVAS PDF report individually and consolidated the results into a structured, scanner-independent schema.

The dataset is available in multiple formats: **JSON**, **JSONL**, **CSV**, **XLSX**, enabling flexible use for benchmarking, validation, research, and integration with different tools and workflows.

## Dataset Files

The main dataset files are:

- `dataset_<timestamp>.json` — Full dataset in JSON format
- `dataset_<timestamp>.jsonl` — Dataset in JSON Lines format (one record per line)
- `dataset_<timestamp>.csv` — Dataset in CSV format
- `dataset_<timestamp>.xlsx` — Dataset in Excel format

### Dataset Fields

Each vulnerability record in the dataset contains the following fields:

| Field                    | Description                                       |
| ------------------------ | ------------------------------------------------- |
| id                       | Unique identifier for the vulnerability           |
| Name                     | Vulnerability name                                |
| description              | Detailed description of the vulnerability         |
| detection_result         | Result of the vulnerability detection             |
| detection_method         | Method used to detect the vulnerability           |
| impact                   | Potential impact of the vulnerability             |
| solution                 | Recommended solution or mitigation                |
| insight                  | Additional insights about the vulnerability       |
| product_detection_result | Product detection information                     |
| log_method               | Logging method used                               |
| cvss                     | CVSS score (0.0–10.0)                             |
| port                     | Affected port number                              |
| protocol                 | Protocol (e.g., tcp, udp)                         |
| severity                 | Severity level (Critical, High, Medium, Low, Log) |
| references               | CVE IDs, URLs, and other references               |
| plugin                   | Plugin identifier                                 |
| plugin_details           | Additional plugin details                         |
| instances                | Number of instances found                         |
| report                   | Name of the source report                         |
| source                   | Scanner source (e.g., OPENVAS)                    |

## Extraction Metrics and Coverage

The dataset contains a total of **6,700 vulnerabilities**, all extracted from 129 OpenVAS PDF reports generated by scanning Docker containers. For coverage evaluation, a CSV file exported from OpenVAS ("vulnnet_scans_openvas.csv") was used as the baseline, containing **6,343 vulnerabilities**.

### Extraction Metrics

- **Total vulnerabilities in baseline (OpenVAS CSV):** 6,343
- **Total vulnerabilities extracted:** 6,700
- **Recall:** 6,101 of 6,343 (0.9618) — coverage: proportion of real vulnerabilities extracted
- **Invented (not in baseline):** 599 (8.94%) — vulnerabilities present in the dataset but not in the baseline
- **Missing (in baseline but not in dataset):** 242 (3.82%) — vulnerabilities not extracted
- **Precision:** 0.9106 — proportion of extracted vulnerabilities that are correct
- **F1-score:** 0.9355 — harmonic mean between precision and recall

### Severity Distribution

| Severity  | Count | % of Total |
| --------- | ----- | ---------- |
| Critical  | 964   | 14.39%     |
| High      | 1,465 | 21.87%     |
| Medium    | 1,908 | 28.48%     |
| Low       | 494   | 7.37%      |
| Log       | 1,869 | 27.90%     |
| **Total** | 6,700 | 100%       |

This distribution shows a heterogeneous and representative dataset, with a wide range of severities from low-impact alerts to high-criticality vulnerabilities, reflecting the diversity of real-world production environments.

### Methodology

Coverage and accuracy were estimated by comparing the extracted dataset against the OpenVAS CSV baseline using fuzzy matching (threshold 85%) on vulnerability names and host/IP. No textual similarity metrics (BERTScore, ROUGE-L) were applied due to schema differences; the CSV was used exclusively for count-based accuracy, with correspondence established via fuzzy matching.

## Dataset overview

The generated metadata file (`metadata.xlsx`) contains all report names used in the dataset creation, as well as the number of vulnerabilities found in each report and their severity counts. This file provides a complete overview of the reports and their extracted vulnerability statistics.

| report  | vulnerability_count | Critical | High | Medium | Low | Log | source  |
| ------- | ------------------- | -------- | ---- | ------ | --- | --- | ------- |
| Example | 537                 | 79       | 126  | 236    | 30  | 66  | OPENVAS |

- **report**: The name or identifier of the scanned report.
- **vulnerability_count**: Total number of vulnerabilities found in the report.
- **Critical, High, Medium, Low, Log**: Number of vulnerabilities per severity level.
- **source**: The scanner used for extraction (e.g., OPENVAS).

## Extraction Comparison

The extraction comparison files (`reports/extraction_comparison.txt` and `.xlsx`) are generated by comparing the extracted dataset against the baseline (`reports/vulnnet_openvas_17230143.csv`). The `.txt` file shows detailed metrics such as recall, precision, F1-score, and examples of invented and missing vulnerabilities. The `.xlsx` file contains two sheets:

- **non-existent**: lists all vulnerabilities invented by the extraction model (present in the dataset but not in the baseline).
- **absents**: lists all vulnerabilities present in the baseline but missing from the extracted dataset.
