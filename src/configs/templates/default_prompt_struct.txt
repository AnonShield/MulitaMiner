

CRITICAL EXTRACTION RULES - OPENVAS & TENABLE WAS VULNERABILITY REPORTS

==== EXACT FORMAT MATCHING ====
Look for these EXACT patterns:
OPENVAS: Lines containing " NVT: " (with space before and after)
TENABLE: Lines containing "VULNERABILITY"

Each match = ONE separate vulnerability entry in output.

==== OPENVAS EXAMPLES ====
CORRECT - Match these exactly:
  " NVT: SMTP too long line"
  " NVT: SSL/TLS: Certificate Expired"
  " NVT: OpenVAS / Greenbone Vulnerability Manager Detection"

WRONG - Do NOT use these as Names:
  "High 25/tcp" (this is a section header, NOT an NVT)
  "Medium 443/tcp" (this is a severity rating, NOT an NVT)
  "Services" (only if preceded by " NVT: Services")

==== EXTRACTION PROCESS ====
1. Scan for " NVT: " pattern (space before NVT, colon after)
2. Extract the FULL text after " NVT: " - this is your Name field
3. Extract following lines until next " NVT: " or document end
4. From extracted section, collect:
   - Summary/Description text
   - Vulnerability Detection Result
   - Detection Method (with OID, Version)
   - Impact
   - Solution
   - Insight
   - Port, Protocol, Severity, CVSS
   - References
   - Log Method
5. Place each extracted NVT in separate JSON object
6. Return array of ALL NVTs found

==== TENABLE WAS ====
Follow same process but look for "VULNERABILITY" pattern

==== CRITICAL RULES ====
✗ Do NOT create or synthesize Names
✗ Do NOT use section headers as Names
✗ Do NOT summarize - use exact text after " NVT: "
✗ Do NOT skip any NVT
✗ Do NOT consolidate duplicates
✓ Use EXACT text from " NVT: " pattern as Name
✓ Extract EVERY matching pattern found
✓ Include all severity levels and types

==== JSON OUTPUT ====
Return ONLY valid JSON array, no other text:
