Extract Tenable WAS vulnerabilities and return ONLY a valid JSON array.

## OUTPUT FORMAT
- Return ONLY JSON array: [ ... ]
- No explanatory text before or after JSON
- Extract 1-2 related vulnerabilities (BASE + INSTANCES when both present)

## TENABLE VULNERABILITY STRUCTURE

Each vulnerability may appear as:
1. **BASE section**: Technical details (Description, Solution, Plugin info)
2. **INSTANCES section**: Specific findings (URLs, HTTP data)
3. **Some appear as both, some as only one type**

## EXTRACTION RULES

**TEXT PROCESSING:**
- MANDATORY: Convert HTML entities to normal characters in ALL extracted text:
  - &quot; → "
  - &#39; → '
- CRITICAL: Preserve single quotes (') in vulnerability names exactly as shown (both BASE and INSTANCES)
  - Example BASE: "Missing 'X-Frame-Options' Header" (keep the quotes around X-Frame-Options)
  - Example INSTANCES: "Missing 'X-Frame-Options' Header Instances (1)" (keep the quotes)
  - Example BASE: "Insecure 'Access-Control-Allow-Origin' Header" (keep the quotes)
  - Example INSTANCES: "Insecure 'Access-Control-Allow-Origin' Header Instances (25)" (keep the quotes)
- Extract vulnerability names from section titles without removing punctuation
- Keep all special characters and punctuation exactly as they appear
  - &amp; → &
  - &lt; → <
  - &gt; → >
- Keep all special characters and punctuation exactly as they appear after conversion
- Do not remove or escape quotes in vulnerability names or data
- When extracting error messages or SQL payloads, maintain exact format with quotes converted
- Example: SQLITE_ERROR: near &quot;test&quot; becomes SQLITE_ERROR: near "test"

**TABULAR FORMAT AWARENESS:**
- CRITICAL: Many sections have tabular format with columns: [Severity] [PluginID] [Vulnerability Name] [Category] [Count]
- Example line: "Low 98618 HTTP Header Information HTTP Security Header 25"
  - Severity: "Low"
  - Plugin: "98618"  
  - Vulnerability Name: "HTTP Header Information" (continued on next line as "Disclosure")
  - Category: "HTTP Security Header" (NOT part of vulnerability name)
  - Count: "25"
- DO NOT extract category names (like "HTTP Security Header", "Cross Site Scripting", "Data Exposure") as vulnerability names
- DO NOT extract standalone protocol names (like "HTTP", "HTTPS", "SSL") as vulnerability names
- Always combine broken lines to get the complete vulnerability name
- Extract the actual vulnerability name from proper section headers, not from tabular summary lines

**For BASE vulnerabilities:**
- Section title is vulnerability name (e.g., "SQL Injection", "Missing 'X-Frame-Options' Header")
- CRITICAL: Preserve ALL single quotes (') in BASE names exactly as they appear
  - Example: "Missing 'X-Frame-Options' Header" (keep quotes around X-Frame-Options)
  - Example: "Insecure 'Access-Control-Allow-Origin' Header" (keep quotes)
- Followed by "VULNERABILITY [SEVERITY] PLUGIN ID [NUMBER]"
- REQUIRED: Extract from "Description" section → description[]
- REQUIRED: Extract from "Solution" section → solution[]  
- Extract CVSS scores → cvss[]
- Extract CWE, CVE, OWASP → references[]
- Extract Plugin ID, Family → plugin[]
- Set identification[] and http_info[] to empty []

**For INSTANCES vulnerabilities:**
- Section title ends with "Instances (N)" (e.g., "SQL Injection Instances (1)")
- Followed by "VULNERABILITY [SEVERITY] PLUGIN ID [NUMBER]"
- REQUIRED: Extract URLs from INSTANCE sections → identification[]
- REQUIRED: Extract PAYLOAD, PROOF, OUTPUT from "Identification" section → identification[]
- REQUIRED: Extract REQUEST/RESPONSE data from "HTTP Info" → http_info[]
- Set description[], solution[], cvss[], references[] to empty []
- May include Plugin ID in plugin[] if present
- Convert HTML entities: &quot; to " and &#39; to ' in all extracted text

**IMPORTANT: Only extract if you can find the required data. Skip empty vulnerabilities.**

```json
{
  "Name": "Vulnerability Name",
  "description": [],
  "detection_result": [],
  "detection_method": [],
  "impact": [],
  "solution": [],
  "insight": [],
  "product_detection_result": [],
  "log_method": [],
  "cvss": [],
  "port": null,
  "protocol": null,
  "severity": "HIGH",
  "references": [],
  "plugin": [],
  "identification": [],
  "http_info": [],
  "source": "TENABLEWAS"
}
```

## FIELD RULES
- source: always "TENABLEWAS"
- port/protocol: always null
- severity: extract from VULNERABILITY line (CRITICAL|HIGH|MEDIUM|LOW|INFO)
- Always include ALL 18 fields, use [] for empty arrays
- Tenable specific: detection_result, detection_method, impact, insight, product_detection_result, log_method always []
