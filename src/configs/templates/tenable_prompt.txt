Extract Tenable WAS vulnerabilities and return ONLY a valid JSON array.

## OUTPUT FORMAT
- Return ONLY JSON array: [ ... ]
- No explanatory text
- All strings in "double quotes"
- Empty arrays use []
- Complete extraction (no truncation)

## TENABLE WAS VULNERABILITY TYPES

Tenable WAS has TWO types per vulnerability:

**INSTANCES** - URLs and HTTP data
- Pattern: "Vulnerability Name Instances (N)"
- Contains: identification[], http_info[], severity

**BASE** - Technical details  
- Pattern: "Vulnerability Name" (no Instances suffix)
- Contains: description[], solution[], cvss[], references[], plugin[]

## EXTRACTION RULES

**For INSTANCES:**
- Get name from section headers only (keep "Instances (N)")
- Extract ALL URLs in identification[]
- Extract ALL HTTP data in http_info[]
- Include REQUEST/RESPONSE headers line by line

**For BASE:**
- Get name from section headers (remove "Instances (N)" if present)
- Extract from Description section → description[]
- Extract from Solution section → solution[]
- Extract CVSS scores and vectors → cvss[]
- Extract CWE, CVE, OWASP references → references[]
- Extract Plugin ID, Family → plugin[]

## TENABLE NAME VALIDATION

**REJECT as invalid names:**
- "VULNERABILITY [SEVERITY] PLUGIN ID [NUMBER]"
- Page numbers, timestamps, headers
- HTTP status codes (HTTP/2 500, HTTP/2 400)
- HTTP headers (access-control-allow-origin)
- URLs or "Page [URL] has no [header] defined"

**Transform descriptive text to proper names:**

❌ "Page https://site.com has no X-Frame-Options header defined"  
✅ "Missing 'X-Frame-Options' Header Instances (1)"

## TENABLE JSON STRUCTURE (18 fields required)

```json
{
  "Name": "Vulnerability Name",
  "description": [],
  "detection_result": [],
  "detection_method": [],
  "impact": [],
  "solution": [],
  "insight": [],
  "product_detection_result": [],
  "log_method": [],
  "cvss": [],
  "port": null,
  "protocol": null,
  "severity": "HIGH",
  "references": [],
  "plugin": [],
  "identification": [],
  "http_info": [],
  "source": "TENABLEWAS"
}
```

## FIELD RULES
- source always "TENABLEWAS"
- port/protocol always null
- severity: CRITICAL|HIGH|MEDIUM|LOW|LOG
- Unused fields always empty: detection_result=[], detection_method=[], impact=[], insight=[], product_detection_result=[], log_method=[]
