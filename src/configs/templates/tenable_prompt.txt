🔸 Name pattern: "Vulnerability Name Instances (N)"

TYPE 2: BASE VULNERABILITIES (Descriptions and Solutions)  
🔸 Name pattern: "Vulnerability Name" (no "Instances" suffix)
🔸 Contains Description, Solution, Plugin Details
🔸 Contains Risk Information with CVSS scores
🔸 Focus: description, solution, plugin, cvss, references arrays

EXTRACTION REQUIREMENT:
✅ Extract ALL Instances vulnerabilities (Type 1)
✅ Extract ALL Base vulnerabilities (Type 2)
✅ Output both types in same JSON array
✅ Match by same base name and plugin ID

INSTANCES VULNERABILITIES EXTRACTION (Type 1):

STEP 1 - IDENTIFY INSTANCES BLOCKS:
🔹 Look for headers containing "Instances (N)" pattern
🔹 Find "VULNERABILITY [SEVERITY] PLUGIN ID [number]" lines
🔹 Extract from INSTANCE markers and Identification sections

EXAMPLES:
✅ "PHP Unsupported Version Instances (14)"
✅ "Missing HTTP Strict Transport Security Policy Instances (25)"  
✅ "Directory Listing Instances (8)"
✅ "Apache 2.4.x < 2.4.41 Multiple Vulnerabilities Instances (1)"

STEP 2 - EXTRACT INSTANCES DATA:
🔹 Name: Keep FULL name including "Instances (N)" suffix
🔹 severity: From "VULNERABILITY [SEVERITY] PLUGIN ID" line
🔹 identification: Extract ALL Identification sections found in the vulnerability - COMPLETE content from EVERY Identification section (descriptive text AND URLs as separate array elements)
🔹 http_info: Extract ALL HTTP Info sections found in the vulnerability - COMPLETE content from EVERY HTTP Info section (REQUEST MADE, REQUEST HEADERS, RESPONSE HEADERS as separate array elements)

⚠️ CRITICAL FOR INSTANCES: Both identification and http_info are ARRAYS - extract ALL occurrences, not just the first one found!

STEP 3 - BUILD INSTANCES JSON:
🔹 Fill: identification[] (ALL Identification content from ALL sections: text + URLs), http_info[] (ALL HTTP data from ALL sections), severity
🔹 Empty: description=[], solution=[], plugin=[], cvss=[], references=[
🔹 Empty OpenVAS fields: detection_result=[], detection_method=[], impact=[], insight=[], product_detection_result=[], log_method=[]
🔹 Set: source="TENABLEWAS"

BASE VULNERABILITIES EXTRACTION (Type 2):

STEP 1 - IDENTIFY BASE BLOCKS:
🔹 Look for vulnerability headers WITHOUT "Instances (N)" suffix

STEP 2 - EXTRACT BASE DATA:
🔹 Name: Base vulnerability name WITHOUT "Instances (N)" suffix
🔹 severity: From "VULNERABILITY [SEVERITY] PLUGIN ID" line

🔹 solution: From Solution section (remediation steps)
🔹 plugin: From Plugin Details section (ID, Family, dates)

  • CWE references (CWE-XXX)
  • WASC references (WASC-XX)
  • OWASP references
  • CVE references (CVE-YYYY-XXXXX)
  • BID references
  • URLs
