🎯 TENABLE WAS EXTRACTION LOGIC

Tenable WAS has TWO vulnerability types per issue:

TYPE 1: INSTANCES (URLs + HTTP data)
• Pattern: "Vulnerability Name Instances (N)"
• Contains: URL lists, HTTP requests/responses
• Fill: identification[], http_info[], severity

TYPE 2: BASE (Technical details)  
• Pattern: "Vulnerability Name" (no Instances suffix)
• Contains: Description, Solution, CVSS, References
• Fill: description[], solution[], cvss[], references[], plugin[]

CONSOLIDATION RULES FOR INSTANCES:
🔥 When multiple "Vulnerability Name Instances (N)" with different N numbers exist:
   ✅ Use the HIGHEST N number for final consolidation
   ✅ Example: "Missing 'X-Content-Type-Options' Header Instances (5)" + "Missing 'X-Content-Type-Options' Header Instances (12)" → Keep "Missing 'X-Content-Type-Options' Header Instances (12)"
   ✅ Merge ALL URLs and HTTP info from all instances
🔥 ALWAYS preserve apostrophes/quotes in names during consolidation

EXTRACTION STRATEGY:
1. Find INSTANCES sections → Extract URLs and HTTP data
2. Find BASE sections → Extract descriptions and solutions
3. Output BOTH types in same JSON array
4. Match by base vulnerability name
