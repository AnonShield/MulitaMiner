TENABLE WAS EXTRACTION RULES
═══════════════════════════════════════════════════════════════════════════════════

Tenable WAS has TWO types of vulnerabilities that must be extracted separately:

TYPE 1: INSTANCES VULNERABILITIES (URLs and HTTP info)
🔸 Name pattern: "Vulnerability Name Instances (N)"
🔸 Contains INSTANCE blocks with URLs
🔸 May contain HTTP Info sections

TYPE 2: BASE VULNERABILITIES (descriptions and solutions)  
🔸 Name pattern: "Vulnerability Name" (no "Instances" suffix)
🔸 Contains Description, Solution, Plugin Details
🔸 Contains Risk Information with CVSS scores

INSTANCES EXTRACTION (Type 1):
🔹 Find headers containing "Instances (N)" pattern
🔹 Extract ALL URLs from INSTANCE blocks and Identification sections
🔹 Extract HTTP technical data from HTTP Info sections
🔹 Keep full name including "Instances (N)" suffix

BASE EXTRACTION (Type 2):
🔹 Find vulnerability headers WITHOUT "Instances (N)" suffix
🔹 Extract description from Description section
🔹 Extract solution from Solution section
🔹 Extract plugin metadata from Plugin Details
🔹 Extract CVSS scores from Risk Information
🔹 Extract references from Reference Information

TENABLE JSON OBJECT RULES:
🔹 For INSTANCES: Fill identification and http_info arrays, empty description/solution/plugin
🔹 For BASE: Fill description/solution/plugin arrays, empty identification/http_info
🔹 Set OpenVAS-only fields to empty: detection_result: [], detection_method: [], etc.
🔹 Set source: "TENABLEWAS"
