═══════════════════════════════════════════════════════════════════════════════════
SCANNER-SPECIFIC MAPPING
═══════════════════════════════════════════════════════════════════════════════════

TENABLE WAS (Web Application Scanner):
- Vulnerability format: Heading + "VULNERABILITY [SEVERITY] PLUGIN ID [ID]"
- Asset: From scan target URL (extract domain, IP if visible)
- Severity: From VULNERABILITY marker
- Port: Usually 443 (https) or 80 (http)
- Protocol: http or https from target URL
- Plugin ID: Extract from report

OPENVAS:
- Vulnerability format: "NVT: [name]" or plugin finding
- Asset: From target IP/hostname in scan header
- Severity: From scan results section
- Port: From results section (various ports)
- Protocol: tcp, udp, ssh, smtp, http, https, etc
- OID: Used as plugin ID

═══════════════════════════════════════════════════════════════════════════════════
EXTRACTION RULES
═══════════════════════════════════════════════════════════════════════════════════

1. IDENTIFY VULNERABILITY BLOCKS
   ✅ Each vulnerability has a HEADING followed by details
   ✅ Find patterns: "VULNERABILITY [SEVERITY] PLUGIN ID" or "NVT: [name]"
   ✅ Headings are SHORT (2-15 words) before severity/plugin markers

2. REJECT THESE PATTERNS
   ❌ "Access to .htaccess file", "Access to .htpasswd file"
   ❌ Anything starting with: "The", "No ", "Access to", "File", "Page"
   ❌ Anything containing URLs or paths
   ❌ From description/output sections, only from headings

3. EXTRACT FIELDS EXACTLY
   ✅ Use exact text from headings (preserve capitalization)
   ✅ Severity: Standardize to CRITICAL, HIGH, MEDIUM, LOW, INFO
   ✅ Port: Extract as number (443, 22, 80, etc) or null if unknown
   ✅ Protocol: Lowercase (https, http, tcp, ssh, smtp, etc)
   ✅ CVE format: CVE-YYYY-XXXXX (e.g., CVE-2023-12345)
   ✅ CVSS scores: Numeric only (7.5, not "CVSS: 7.5")

4. ASSET INFORMATION
   - Extract from scan target and report header
   - domain: Extract from URL or FQDN field
   - IP address: From target or header section
   - OS: From system information section if present
   - Type: "Web Application" for web scans, "Network Host" for network scans

5. INSTANCES HANDLING
   If same vulnerability affects multiple items:
   - Create SEPARATE object for each instance
   - shared fields (definition.*) are identical
   - instance fields (asset, port, output) differ
