**🚨 MOST IMPORTANT RULE - HOW TO FIND VULNERABILITY NAMES:**

The vulnerability Name is the **SINGLE LINE immediately before** "VULNERABILITY [SEVERITY] PLUGIN ID [number]"

**CRITICAL - INSTANCE blocks:**
For Tenable WAS, only extract Instances when the word INSTANCE and its associated evidence (such as URLs or paths) actually appear in the PDF text. Do not invent, infer, or summarize Instances. Each Instance must correspond to a real, explicit occurrence in the report. Only extract an 'Instances (N)' block for a Name if the exact text 'Instances (N)' appears in the PDF, and ensure each Name has at most one unique Instances block. Do not create duplicate or erroneous Instances blocks.

Example:
```
Interesting Response Instances (25)          ← Name (ONE vulnerability)
VULNERABILITY INFO PLUGIN ID 98050
INSTANCE
https://.../passwords/.htpasswd              ← Evidence (goes in identification[])
INSTANCE
https://.../passwords/.htaccess              ← Evidence (goes in identification[])
```

**REJECTION PATTERNS - NOT valid Names:**

❌ **Pattern 1:** Contains "https://", "http://", "://", "www.", "CVE-", "function()", "method()"

❌ **Pattern 2:** Begins with "Page ", "The ", "A ", "An ", "No ", "This ", "It ", "There " OR contains "The scanner did not find"

❌ **Pattern 3:** Contains "has no", "detected on", "were found", "have been", "was detected", "is missing", "should be" OR starts with numbers like "2 HTML comments"

❌ **Pattern 4:** Contains ": " (colon-space) OR matches "HTTP/[0-9]", "GET ", "POST ", "TRACE ", "DEBUG "

❌ **Pattern 5:** Starts with "Access to", "Path to", "File ", "Directory " OR contains "/.ht", "/.git", "/web.", "/config", "/install"

❌ **Pattern 6:** Ends with " flaw", " vulnerability", " issue", " weakness", " bug" OR contains "pointer", "buffer", "overflow", "injection", "bypass" (when alone)

✅ **VALID Names:**
- Short (2-10 words), capitalized heading before "VULNERABILITY [SEVERITY] PLUGIN ID"
- May end with "Instances (N)"
- Examples: "Missing 'X-Frame-Options' Header", "Apache 2.4.x < 2.4.60 Multiple Vulnerabilities"

**CVE HANDLING - "Multiple Vulnerabilities" = ONE object:**

When Name contains "Multiple Vulnerabilities", extract ONLY that Name with ALL bullets in description[]

Example:
```
Apache 2.4.x < 2.4.60 Multiple Vulnerabilities    ← Name (ONE vulnerability)
VULNERABILITY CRITICAL PLUGIN ID 114360
Description
- Serving WebSocket protocol upgrades... (CVE-2024-36387)     ← Bullet in description[]
- SSRF in Apache HTTP Server... (CVE-2024-38472)              ← Bullet in description[]
- Encoding problem in mod_proxy... (CVE-2024-38473)           ← Bullet in description[]
```

✅ CORRECT: ONE object with all bullets in description[], CVEs in references[]

❌ WRONG: Multiple objects with bullet fragments as Names like:
- "WebSocket Protocol Upgrade Null Pointer Dereference"
- "SSRF NTML Hash Leakage on Windows"
- "mod_proxy Encoding Authentication Bypass"
